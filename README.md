# Hirebot 🤖

Локальный инструмент для поиска работы (PM / Product / CTO) через hh.ru: умные автоотклики, персональные сопроводительные письма и аналитика воронки.

Всё хранится локально — SQLite-файл `prisma/dev.db`, никаких внешних серверов.

📋 **Полный план трудоустройства** (резюме под ATS, hh, LinkedIn, фото, ежедневная рутина, собеседования) — в [`ROADMAP.md`](ROADMAP.md).

## Возможности

- **Поиск и скоринг вакансий** — каждая вакансия оценивается 0–100 по релевантности роли, зарплате, свежести, признакам агентства. Лимит hh — 200 откликов/день, поэтому откликаемся только на лучшее.
- **Массовый автоотклик** — кнопка «Запустить»: бот сам проходит очередь вакансий со скорингом выше порога и шлёт отклики до дневного лимита, с живым прогрессом и логом. Вакансии с обязательным тестом автоматически уходят в список «только вручную».
- **Сопроводительные письма** — генерация под конкретную вакансию через установленный терминальный Claude Code (`claude -p`, работает по подписке, API-ключ не нужен) или по шаблонам с плейсхолдерами `{position}` / `{company}`. Перед отправкой — автоматический чеклист качества (длина, персонализация, клише, призыв к действию).
- **Список «только вручную»** — вакансии с тестом: генерируешь письмо, копируешь, переходишь на hh по ссылке, откликаешься сам и жмёшь «Отметить отправленным» — они тоже попадают в воронку.
- **Воронка** — статусы подтягиваются с hh (просмотрен / приглашение / отказ), дальше двигаешь вручную: скрининг → собеседование → финал → оффер.
- **A/B-аналитика** — конверсия в приглашение по версиям резюме и типам писем: видно, что реально работает.
- **Чёрный список** работодателей и скрытие мусорных вакансий.

## Запуск

```bash
npm install
npx prisma db push   # создаёт prisma/dev.db
npm run dev          # http://localhost:3000
```

> ⚠️ Windows: запускай команды из пути с правильным регистром (`WebstormProjects`, не `webstormprojects`) — иначе сборка Next падает на дублировании модулей.

## Настройка

1. **ngrok** (hh.ru не принимает localhost как Redirect URI):
   аккаунт на [ngrok.com](https://ngrok.com) → `ngrok config add-authtoken <токен>` → `ngrok http 3000`.
   Полученный адрес вида `https://xxxx.ngrok-free.app` понадобится на следующем шаге.
2. **hh.ru**: [dev.hh.ru/admin](https://dev.hh.ru/admin) → «Добавить приложение»,
   Redirect URI: `https://xxxx.ngrok-free.app/api/auth/hh/callback`.
   Client ID, Secret и тот же Redirect URI впиши в `.env` (`HH_REDIRECT_URI`).
3. **Письма**: ничего настраивать не нужно — используется установленный Claude Code (`claude -p`) по твоей подписке.
4. Перезапусти dev-сервер, открой «Настройки» → «Подключить hh.ru» → «Синхронизировать резюме».
5. Каждому резюме назначь роль (PM / Product / CTO) — под неё работает скоринг.

> ngrok нужен только в момент OAuth-авторизации (callback). Работать с ботом продолжаешь на `http://localhost:3000`.

## Рабочий цикл

1. «Вакансии» → поиск → бот сохраняет и оценивает найденное.
2. «Массовый автоотклик» → задаёшь порог скоринга и лимит → «Запустить». Бот шлёт отклики один за другим (с персональными письмами от Claude или по шаблону — на выбор) и останавливается на дневном лимите hh.
3. Вкладка «Только вручную (тест)» — по этим вакансиям: «Письмо» → «Скопировать» → «Откликнуться на hh ↗» → после отклика «Отметить отправленным».
4. Раз в день на дашборде «Обновить статусы с hh.ru».
5. Приглашения двигаешь по воронке вручную; смотришь конверсии по резюме и типам писем — и докручиваешь.

## Ограничения hh API

- 200 откликов в сутки на аккаунт.
- Вакансии с обязательным тестом/опросником — только ручной отклик (помечены в списке).
